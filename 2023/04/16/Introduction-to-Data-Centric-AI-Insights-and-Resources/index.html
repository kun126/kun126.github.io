Introduction to Data-Centric AI: Insights and Resources

Kun Chen
2023-04-16

As someone who has always been fascinated with the field of data science, I was particularly drawn to this topic after having access to real-world data during my internships. I realized that how messy and insufficient data can be, which posed a challenge for advanced ML models. From my experience, I also found that interpretation can sometimes carry more weight than accuracy, which led to a sense of gap between course projects and real-world applications. No matter which model I chose, I always felt that I needed to invest a massive amount of time in improving the quality of data and features, which is a crucial part of data-centric AI.

That’s why I’m excited to share that there’s a new online course available this year that focuses specifically on the data perspective of ML - Introduction to Data-Centric AI. In this post, I’ll be sharing some valuable insights and resources from the course.

Differences between Data-centric AI and Model-centric AI

Traditionally, model-centric AI is the dominant approach, where the focus is on producing the best model by tuning hyperparameters, adjusting the loss function, and regularization. In this approach, the dataset is assumed to be fixed and curated. However, in the real world, datasets are often messy, and improving models is not always the best way to achieve better performance.

This is where data-centric AI comes in. It provides an alternative approach by improving the ML data itself, rather than just the model. Data-centric AI involves two forms:

  • AI algorithms that use their understanding of data to improve the training process, i.e., Curriculum learning.
  • AI algorithms that modify data to improve model results, i.e., Confident learning.

An Effective Data-centric AI Pipeline

  1. Explore the data, fix fundamental issues, and transform it to be ML appropriate.
  2. Train a baseline ML model on the properly formatted dataset.
  3. Utilize this model to help you improve the dataset (techniques taught in this class).
  4. Try different modeling techniques to improve the model on the improved dataset and obtain the best model.

Note: Step 3 and step 4 can be iterated multiple times to achieve the best possible results.

Methodologies for Data-centric AI: Techniques and Tools

Label error detection with confidence learning

Confidence learning is a type of learning that was developed for image classification with the goal of identifying label errors that are independent of the classification model used. The approach is based on the assumption that label noise is class-conditional, meaning that it only depends on the latent true class and not the data itself. Here are the main steps involved in confidence learning:

  1. Self-confidence for each class: Compute a class threshold for each class by averaging the out-of-sample predicted probabilities of samples labeled with that class.
  2. Confident joint: Construct a noisy label x latent label matrix that counts label errors for each sample. The diagonal cells record correct labels, while off-diagonal cell [i, j] will add one when an image with a given label class[i] has the highest predicted probability of class[j], which should also exceed the class threshold of j. The sum of off-diagonals is k.
  3. Self-confidence for each sample: Extract the predicted probability for a sample’s given label. Rank them to find k less confident ones as the error labels. Alternatively, you can use normalized margin or confidence weighted entropy. For more information, please refer to cleanlab.find_label_issues.

A simple method for handling label errors is to drop them and retrain the classifier. However, this approach can be overly simplistic and can lead to a loss of valuable data. In real-world applications, a more effective approach is to have humans review the issues and correct the labels.
